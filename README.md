# ⚡ 智行新能源车服 AI 管家 (Smart EV Service AI Assistant)

这是一个基于大语言模型 (LLM) 和 RAG (检索增强生成) 技术打造的新能源汽车智能管家应用。不同于目前随处可见的“死板”基础问答机器人，本项目通过 LangChain 搭建的 ReAct 架构，让 AI 能够按照一定的思考逻辑链（发现问题 -> 寻找工具 -> 查询数据 -> 综合解答）自主调用外部查询工具。它无缝融合了真实的车辆运行遥测数据与详尽的新能源知识库，能在售前或售后场景中为车主提供高度定制化的交互体验。

---

## 🌟 核心与技术亮点

*   **真正的 Agent 智能分发**：AI 不再只是个“文科生”。它被配置了强大的心脏（ReAct Agent），可以通过上下文自主判别什么时候该使用计算工具，什么时候该调取后台数据库查车辆 SOH（健康度）损耗，什么时候该去挂载最新的天气数据。
*   **垂直领域 RAG 知识检索**：在 `data/` 目录中下沉了专门针对新能源汽车的重度垂直语料（涵盖《极寒炎热天气电池保养指南》、《高压系统故障盲操排查手册》等）。借由 Chroma 向量数据库进行高维检索，彻底杜绝了通用大模型在专业汽车机械、三电理论上容易产生的“幻觉”。
*   **纯手写沉浸式前端交互**：抛弃了千篇一律且难以高度定制的 Streamlit / Gradio 懒人模板。整体前端由纯原生 HTML5/JS 结合高阶 CSS 手工雕琢而成。深度定制了目前极为前沿的**高级暗黑毛玻璃 (Premium Glassmorphism)** 拟态 UI，并利用 Server-Sent Events (SSE) 取代 RESTful，完美还原了丝滑的逐字打印流式响应与呼吸感知特效。
*   **车况数据沙盘模拟引擎**：后台开发了基于 `csv` 构建的沙盒运行数据处理链，涵盖了不同测试车编队 (如 VIN1001-1010) 长达一年的动态衰减和能耗变化特征模拟，支撑起极度写实的“基于数据的报告总结”。

---

## 🏗️ 架构概览

*   **AI 调度中枢**：`LangChain` 框架级支持应用。
*   **执行思维轴**：大模型基座搭配 `ReAct (Reason and Act)` Prompt 配置。
*   **向量记忆节点**：`ChromaDB`，配合 `MD5` 哈希特征对比的增量更新机制，只对发生变化的数据源文件做重新 Chunk 与 Embedding 动作，大幅节约本地化算力。
*   **高性能后端接口**：放弃同步阻塞型服务，依托 `FastAPI` 和 `Uvicorn` 构筑纯异步非阻塞的后台大流量流式 (`StreamingResponse`) 输出。
*   **无状态前端展现**：原生原生栈构筑，内置智能 DOM 记忆切片处理（侧边栏自由无缝切换不同账户 VIN 数据并自动重绘界面渲染），彻底做到轻量、可扩展。

## 🛠️ 本地部署与运行

开发此项目时使用的是 Python 3.10+ 环境。克隆到本地后的运行极其简单：

```bash
# 1. 命令行进入项目根目录
cd Agent项目

# 2. 激活您的专属环境
conda activate Rag
# 如果缺少依赖，请执行 pip install fastapi uvicorn langchain chromadb 等依赖 (或依靠 requirements.txt)

# 3. 驱动服务
# 我们采用 uvicorn 作为 ASGI 服务器进行带有热重载的启动
uvicorn api:app --reload
```
最后，只需在浏览器中打开 `http://127.0.0.1:8000` 即可与您私人的 AI 车服管家展开对话。

## 📝 开发者碎碎念

起初搭这个玩意只是一心想弄明白 Agent 这种带思考的架构挂载多个外部 API 时到底是怎么跑通链路的。但随着工具链的堆砌，我发现把它代入到“新能源汽车售后管家”这个真实场景里格外的契合（由于新能源汽车本身就是个跑在公路上插满传感器和后台监控的巨大智能硬件）。

在这个项目里亲手踩过了很多坑：比如大模型常常会在调用组合工具后陷入“无限循环思考”而忘记最开始的主线任务（通过打磨更严谨的 Middleware 解决）；再比如使用 FastAPI 配合前台原生 Fetch 读取流时令人抓狂的截断渲染乱码（最终通过严谨的分片机制锁死更新解决）。

最终把这样一个高颜值、逻辑自洽、带有一点极客味儿的版本放出来，成就感拉满，它应该能成为后续我自己学习和衍生更多 AI 应用业务时非常优质的一套核心脚手架。
